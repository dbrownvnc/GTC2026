01 — 배경HPC의 역사: 하드웨어-소프트웨어 공진화

고성능 컴퓨팅(HPC, High Performance Computing)은 오랫동안 하드웨어 발전에 맞춰 소프트웨어가 진화하고, 소프트웨어의 요구가 다시 하드웨어를 자극하는 공진화(co-design) 패턴을 반복해왔습니다.

"Moore의 법칙이 끝나가던 시점에 AI가 찾아왔다. AI 워크로드는 높은 정밀도를 요구하지 않았고, 이는 혼합 정밀도의 폭발적 발전으로 이어졌다."

— GTC 2026 S81792 발표자

무어의 법칙(Moore's Law)이란 반도체 집적회로의 트랜지스터 수가 2년마다 2배씩 증가한다는 경험적 법칙입니다. 이 법칙이 한계에 달하자 HPC 커뮤니티는 새로운 최적화 방법을 모색했고, AI와의 융합이 자연스러운 답으로 부상했습니다.

02 — 핵심 개념시뮬레이션의 진화: 실험 → 시뮬레이션 → AI

과거에는 자동차 충돌 안전 테스트를 위해 실제 차량과 센서 달린 마네킹을 부수거나, 항공기 엔진을 풍동(wind tunnel)에 넣어 시험해야 했습니다. 시뮬레이션은 이런 고비용 실험을 가상으로 대체했습니다.

시뮬레이션의 3가지 현대적 역할

▼ 각 역할을 탭하여 자세히 보기

① 전통적 문제 해결
복잡한 실세계 시스템을 수학적으로 모델링하여 비용 효율적으로 분석. 유체역학(CFD), 기후 모델링, 분자 시뮬레이션 등이 대표적 사례입니다.
② AI 학습 데이터 생성기
시뮬레이션을 반복 실행하며 데이터를 변형(perturbation)하여 다양한 시나리오를 대량으로 만들어냅니다. 이 데이터로 AI 모델을 학습시켜 서로게이트(대리) 모델 또는 디지털 트윈을 생성합니다.
③ AI 추론 결과 검증 도구
AI 모델이 빠른 추론 결과를 제공한 뒤, 시뮬레이션이 다시 "그게 정말 정확한가?"를 검증합니다. AI와 시뮬레이션이 상호 보완하는 구조입니다.

03 — 가속 시나리오소프트웨어 최적화의 3가지 경로

시나리오 1 — 내부 가속: CPU → GPU 이전

가장 전통적인 방법입니다. 기존에 CPU에서 실행되던 코드의 루프(loop) 연산을 GPU로 옮겨 더 높은 병렬성을 확보합니다. AMGX(희소 선형 해석기)와 cuDSS(직접 희소 선형 해석기)가 대표적인 라이브러리입니다.

📌 주요 라이브러리

AMGX — 회로 시뮬레이션, CFD(유체역학), 열/저수지 코드 등 다양한 도메인의 희소(sparse) 선형 연립방정식 가속에 사용

cuDSS — 직접(direct) 희소 선형 해석기. 문제 유형에 따라 두 라이브러리를 선택적으로 활용

시나리오 2 — 혼합 정밀도: 텐서코어 활용

AI 워크로드는 낮은 정밀도(precision)로도 우수한 결과를 냅니다. NVIDIA GPU는 FP64부터 FP4까지 다양한 정밀도를 지원하며, 정밀도를 낮출수록 연산 성능이 크게 향상됩니다.

그림 2. GPU 세대별·정밀도별 성능 (TFLOPS, 높을수록 좋음)

시나리오 3 — 워크플로 전환: AI 서로게이트 모델

세 번째 시나리오는 시뮬레이션 자체를 AI 모델 학습의 재료로 삼아 서로게이트(surrogate) 모델 또는 디지털 트윈(digital twin)을 만드는 것입니다.

사례 ① 소분자-단백질 결합 시뮬레이션

기존 방식새로운 방식
입력 데이터 → 시뮬레이션 실행 → 효능 점수 출력시뮬레이션 데이터로 AI 모델 학습 → AI가 즉시 효능 점수 예측
매번 수시간~수일 계산질의 1건당 수 초 이내 응답

사례 ② 수치기상예측(NWP) → Earth-2 AI 모델

WRF(Weather Research and Forecasting) 모델은 주로 Fortran으로 작성되고 MPI+OpenMP를 사용하는 전형적인 HPC 코드입니다. GPU 지원을 위해 OpenACC, std::par, CUDA 등 여러 프로그래밍 모델을 혼용합니다.

WRF 시뮬레이션 출력 → ForecastNet, CorrDiff(Earth-2 내 모델) 학습 데이터로 활용 → 예측 요청 시 시뮬레이션 없이도 빠른 응답 가능.

💡 핵심 인사이트

AI 서로게이트 모델이 확산될수록 원본 시뮬레이션의 속도도 계속 높여야 합니다. 더 빠른 시뮬레이션 = 더 많은 학습 데이터 = 더 좋은 AI 모델이라는 선순환이 만들어지기 때문입니다.

05 — 심층 논의Q&A에서 나온 핵심 쟁점

Q1. 신/구 GPU 세대 간 고정밀도 성능 격차가 왜 낮은가?

FP64 같은 고정밀도에서는 세대 간 성능 차이가 비교적 작은데, 이는 과학 코드들이 실제로는 FP64 연산에서 병목이 걸리는 게 아니라 다른 부분(메모리, 통신 등)에서 병목이 발생하기 때문입니다. 따라서 전체 애플리케이션 관점에서 균형 잡힌 개선을 추구하는 것이 NVIDIA의 설계 철학입니다.

Q2. FP64 에뮬레이션이 과학자들에게 위협적인가?

한 청중이 "미래에 FP64가 사라지는 게 아닌가?"라는 불안을 표명했습니다. 발표자의 답변:

공식 입장

FP64 에뮬레이션은 FP64를 대체하는 게 아니라 텐서코어를 추가로 활용하는 최적화 옵션입니다. B200, Hopper, Vera Rubin 등 컴퓨팅 용도 GPU에는 FP64가 유지됩니다. 에뮬레이션의 실제 장점은 저정밀도 유닛을 쓰므로 메모리 공간도 줄어들어 더 큰 모델을 올릴 수 있다는 점입니다.
