개요

NVIDIA GTC 2026에서 열린 이 패널은 AI 추론(Inference)의 현재와 미래를 다뤘다. 추론이란 이미 학습된 모델이 새로운 데이터를 처리하는 과정으로, 대규모 서비스에서 비용과 속도 모두를 결정짓는 핵심 단계다. 패널리스트들은 처리량 극대화, 초당 토큰 수 향상, 와트당 토큰 효율, 오픈소스 생태계, 주권 AI까지 폭넓게 논의했다.

핵심 주제 1

추론 최적화 기법의 현재와 미래

패널은 현재 추론 최적화의 '나사'들을 열거하며, 이것들이 점점 더 조여질 것이라는 데 모두 동의했다. 최적화는 하드웨어 발전과 알고리즘 설계 두 축 모두에서 동시에 진행된다.

"하드웨어가 발전하고 알고리즘 설계도 발전한다. 두 가지가 함께 올 때, 중간에 양자화 같은 모든 것을 최대한 활용한다."

— Ophir, LTX
양자화 (Quantization) 하드웨어
▾
모델 가중치의 정밀도를 낮추는 기법. FP32 → FP16 → FP8 → FP4(NV FP4)로 진화. 4비트 양자화로도 품질 저하가 매우 적은 수준을 달성했으며, LTX-2(19B 모델)는 6GB VRAM GPU에서도 구동 가능하다는 사례가 소개됐다.
투기적 디코딩 (Speculative Decoding) 알고리즘
▾
한 번에 하나씩 토큰을 예측하는 기존 방식 대신, 작은 드래프트 모델이 여러 토큰을 미리 생성하고 큰 모델이 이를 검증하는 방식. 처리 속도를 크게 높이며, 다양한 양자화 포맷과의 조합 테스트가 숙제로 남아 있다.
KV 캐싱 & 메모리 관리 시스템
▾
시스템 디자인의 캐싱 개념을 AI 추론에 적용. 이미 계산된 Key-Value 값을 재사용해 반복 연산을 줄인다. 텐서를 네트워크를 통해 스트리밍하는 메모리 오프로딩 기법과 결합하면 GPU VRAM이 부족한 환경에서도 대형 모델 구동이 가능하다.
잠재 공간 압축 (Latent Space Compression) 알고리즘
▾
LTX, Black Forest Labs 등 확산(diffusion) 모델 기업들의 핵심 전략. 더 적은 잠재 토큰을 사용해 연산 자체를 줄이는 방식. NVIDIA의 DCGen·DCVideo도 이 방향을 채택했다. 영상 모델에서 특히 효과적이며, 품질과 효율의 균형이 관건이다.
확산 반복 횟수 축소 알고리즘
▾
이미지·영상 생성 확산 모델은 기본적으로 20~30회의 순전파(forward pass)가 필요하다. BFL은 蒸留(distillation) 연구를 통해 이를 4~8회로 줄이는 데 집중하고 있다. VRAM 문제보다 사용자 대기 시간 문제가 더 크기 때문에 반복적이고 빠른 생성이 필수다.

모델 크기: 대형화 vs 소형화

1조 파라미터의 Kimi K2 같은 초거대 모델과, 온디바이스용 초소형 모델이 동시에 주목받고 있다. 패널은 이 두 방향이 수렴하는 것이 아니라 양방향으로 동시에 극단화될 것이라고 진단했다.

소형 모델이 필요한 이유

— Ophir, LTX

핵심 주제 3

지배적 아키텍처: MoE vs Dense

Mistral은 Mixtral 8×7B로 오픈소스 Mixture of Experts(MoE) 모델의 포문을 열었다. 패널은 MoE가 고도의 병렬 서비스 환경에서 압도적인 효율을 보인다고 평가하면서도, 기술적 도전과 새로운 아키텍처의 가능성도 함께 논의했다.

Dense 모델 (밀집 모델)
모든 요청에 전체 파라미터를 활성화한다.

장점: 구현이 단순하고 인프라 요구사항이 낮다. 소규모 배포에 유리.
단점: 파라미터 수가 늘수록 연산 비용이 선형적으로 증가한다. 수천 개의 병렬 요청 처리에 비효율적이다.
Mixture of Experts (전문가 혼합)
전체 파라미터 중 일부 "전문가" 레이어만 선택적으로 활성화한다.

장점: 전체 파라미터는 크지만 실제 연산량은 적다. 병렬 서빙 처리량이 매우 높다. Patrick은 수천 개의 동시 요청 처리 시 MoE가 빛난다고 강조.
단점: 전체 가중치를 메모리에 올려야 해 VRAM 요구량이 크다. 훈련 안정성과 속도 개선이 여전히 연구 중이다.
Diffusion 기반 LM (연구 중)
이미지 생성에서 쓰이는 확산 모델의 원리를 텍스트 생성에 적용한다.

잠재력: 자기회귀(autoregressive) 방식의 대안. 병렬 토큰 생성으로 속도 향상 가능성.
현황: Patrick은 이것이 가장 큰 미지수라고 언급. 아직 LM 수준의 품질에 도달하지 못했으며, 커뮤니티의 전환 여부는 불확실하다.

핵심 주제 4

소버린 AI와 오픈소스의 역할

주권 AI(Sovereign AI)란 국가·기업이 AI 기술과 데이터를 스스로 통제할 수 있는 환경을 의미한다. 패널은 오픈소스가 이를 가능하게 하는 유일한 수단이라고 강조했다.

"우리가 원하는 세계는 모든 국가, 모든 기업이 자신들의 AI를 구축하고 통제할 수 있는 세계다. 그것은 오픈소스로만 가능하다."

— Jeff Boudier, Hugging Face
API 방식 vs 오픈소스 배포의 차이
  • API 방식: 데이터가 외부 서버로 전송됨 → 보안·규제 문제 발생
  • 오픈소스 온프레미스: 가중치를 내부 인프라에 직접 배포 → 데이터 유출 없음
  • 커스터마이징: 오픈 가중치로 자국 문화, 고객 특성에 맞게 파인튜닝 가능
  • vLLM, SGLang: 추론 서버 프레임워크로 효율적 서비스 제공 지원

핵심 주제 6

오픈소스와 비즈니스의 공존

오픈소스 모델을 공개하면서 동시에 수익을 내는 방법에 대해 각 회사는 다른 전략을 구사하고 있다.

Black Forest Labs (BFL)
▾
티어링 전략: API 제공 모델 / 오픈 가중치 모델 / 완전 오픈소스(Apache) 모델로 분리. 오픈소스는 배포(distribution) 전략으로 활용. 커뮤니티에서 LoRA 등 파생 모델이 만들어지면 기업 고객의 관심으로 이어진다.
Hugging Face
▾
프리미엄 플랫폼: 공개 모델은 무료, 조직 보안·추가 컴퓨트·스토리지는 구독제. 300,000개 이상의 조직이 사용. 시애틀 기업 Xad 인수로 AI 네이티브 스토리지 구축 (청크 기반 스토리지로 일반 대비 3배 저렴). 훈련 체크포인트·모델 업로드 대역폭을 최적화.
Mistral AI
▾
솔루션 판매: AI가 창출하는 진짜 가치는 B2B 복잡한 비즈니스 프로세스에 있다고 판단. 오픈소스 모델을 기반으로 기업용 솔루션을 구축·판매. Mistral AI Studio(기업 커스터마이징 플랫폼) 운영. 현장 파견 엔지니어로 고객사 직접 지원.

"아이디어에는 독점이 없다. 오픈소스는 전략이며, 동시에 좋은 비즈니스 전략이 될 수 있다."

— Ophir, LTX
미래 예측

패널리스트의 전망

🎬
실시간 영상 생성 — 3~4년 전만 해도 제대로 동작하지 않았던 영상 모델이 이제 실시간 생성을 목전에 두고 있다. 시각 지능 분야는 텍스트가 이미지·영상의 부분집합이라는 관점에서 폭발적으로 성장할 것이다.
— Tim, Black Forest Labs
🤖
AI가 소프트웨어 개발을 재정의 — 5년 전 "소프트웨어는 머신러닝의 하위 분야"라는 말은 선언적 구호였지만, 지금은 현실이 됐다. AI 퍼스트로 기술 스택 전체가 뒤집히는 패러다임 전환이 진행 중이다.
— Jeff Boudier, Hugging Face
🔊
실시간 스트리밍 음성 AI — 실시간 음성 인식·생성의 오픈소스화, 폰/노트북 온디바이스 구동. 요청-응답 방식에서 스트리밍 입출력 방식으로 인터페이스 패러다임이 바뀐다.
— Patrick, Mistral AI
🕹️
인간-AI 제어의 경계 탐색 — AI에게 어디까지 제어권을 넘기고 어디서 인간이 개입할지 결정하는 것이 가장 중요한 과제. 특히 영상처럼 공간적·시각적 매체에서는 세밀한 제어 인터페이스 설계가 핵심이다.
— Ophir, LTX
