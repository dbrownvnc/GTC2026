정답 및 해설

모든 문항의 정답과 상세 해설입니다.

1
LSST 카메라 해상도
정답: D — 3.2 기가픽셀
189개의 4K×4K CCD 칩을 합산하면 약 3.2 기가픽셀(32억 화소)이 됩니다. 이는 세계 최대의 단일 디지털 카메라입니다. 숫자 189는 구성 칩 수이고, 해상도 자체가 아닙니다.
2
파이프라인 개발 방식
정답: X (거짓)
파이프라인은 약 25년 전 C++98로 시작되었으며, Python 2.3 바인딩이 일부 추가된 형태였습니다. 이후 점차 Python 우선 구조로 마이그레이션하여 현재는 Python 3.14 + C++20 + 일부 Rust 바인딩 구조가 되었습니다.
3
전체 판독 시간
정답: B — 2초
3.2 기가픽셀 전체를 단 2초 만에 읽어냅니다. 이 빠른 판독 속도가 가능해야 500회/야간의 관측 일정을 소화할 수 있습니다.
4
PSF의 의미
정답: 점퍼짐 함수 (Point Spread Function)
PSF(Point Spread Function)는 대기 난류와 광학계에 의해 점광원이 이미지에서 퍼지는 방식을 수학적으로 표현한 합성곱 커널입니다. 대기는 매번 다른 PSF를 만들기 때문에, 차분 이미징 전 두 이미지의 PSF를 일치시키는 작업이 필수적입니다.
5
최대 속도 향상 사례
정답: C — 다이폴 피팅 (멀티 GPU, 5시간→1초)
멀티 GPU 환경에서 다이폴 피팅은 5시간에서 1초로 단축됩니다. 이는 18,000배에 달하는 속도 향상으로, 선택지 중 가장 큰 배율입니다. XFEL은 15시간→0.5초(약 108,000배)가 더 크지만, 이 선택지는 포함되지 않았습니다.
6
GPU 가속 방식
정답: X (거짓)
단순 코드 이식(포팅)이 아니라 알고리즘 자체를 GPU 아키텍처에 맞게 재설계했습니다. 물리를 이해하고, 수학으로 변환하고, GPU 병렬 처리에 최적화된 방식으로 다시 구현한 것이 핵심입니다. 기저 함수 라이브러리도 이 과정에서 새로 개발되었습니다.
7
주요 병목
정답: C — 데이터 입출력(I/O)
프로파일링 결과 I/O가 처리 시간의 약 50%를 차지합니다. 이를 해결하기 위해 HDF5 형식은 80배 최적화되었고, FITS 형식의 GPU 직접 로딩 API를 개발 중입니다. 또한 관측 일정이 사전에 알려져 있어 기준 이미지(Template)를 미리 GPU에 올려두는 방식으로 일부 완화합니다.
8
경보 전송 시스템
정답: Kafka (카프카)
검출된 이벤트는 이미지 절편(Cutout)과 함께 Kafka 메시지 큐에 패키징됩니다. 전 세계 커뮤니티 브로커들이 관심 채널을 구독하여 초신성 브로커, 태양계 브로커 등이 각자 필요한 경보를 수신합니다.
9
HPC-AI 연계 이유
정답: B — AI가 올바른 물리를 수행하는지 검증할 Ground Truth 필요
AI가 빠르게 분류·추론하더라도 그 결과가 물리적으로 정확한지 알 수 없습니다. HPC의 고정밀 수치 계산은 AI 모델에게 "정답"을 제공하고, AI는 이를 바탕으로 학습합니다. 새 데이터가 쌓일수록 파라미터와 오차 계산이 정밀해지는 반복적 개선 루프입니다.
10
다이폴(Dipole)의 의미
정답: 천체가 이동한 양과 방향 (크기·형태 변화 포함)
다이폴은 새 이미지에서 기준 이미지를 뺀 결과물로, 양(positive)과 음(negative) 신호의 쌍으로 나타납니다. 이 쌍의 위치·크기·형태를 분석하면 해당 천체가 얼마나, 어떤 방향으로 이동했는지 알 수 있습니다. 소행성 궤도 예측처럼 정밀도가 생사를 가르는 응용에서 특히 중요합니다.
