객관식 쉬움
이 강연에서 HPC 역사에서 가장 먼저 등장한 소프트웨어 병렬화 기법은 무엇이라고 설명했는가?
AMPI 분산 처리
B스레딩(Threading)
C벡터화(Vectorization)
DGPU 루프 오프로드
정답: C — 벡터화(Vectorization)
발표자는 HPC 병렬화의 첫 번째 단계로 벡터화를 꼽았습니다. 핵심 목표는 "사이클당 더 많은 작업을 처리"하는 것이었으며, 이후 스레딩 → MPI 분산 → GPU 루프 가속 순서로 발전했습니다.

객관식 보통
Blackwell GPU에서 FP64 텐서코어 GEMM과 INT8 텐서코어 성능을 비교했을 때 약 몇 배 차이가 났는가?
A약 5배
B약 10배
C약 30배
D약 100배
정답: C — 약 30배
FP64 텐서코어 GEMM은 150 TFLOPS, INT8 텐서코어는 4,500 TOPS로 약 30배 차이가 납니다. 발표자는 이를 "내 코드에 적용할 수 있다면 엄청난 성과"라고 표현했습니다.

단답형 쉬움
강연에서 소개된 희소(sparse) 선형 연립방정식을 GPU에서 가속하는 라이브러리의 이름을 쓰시오. (대소문자 무관)
정답: AMGX
AMGX(Algebraic MultiGrid on X)는 희소 선형 해석기(sparse linear solver) 라이브러리로, 회로 시뮬레이션·CFD·열 해석·저수지 모델 등 다양한 도메인에서 활용됩니다. cuDSS는 직접(direct) 희소 해석기로 별개의 도구입니다.

객관식 보통
발표자가 "GPU로 처음 이식할 때 가장 먼저 사용하라"고 추천한 프로그래밍 모델은?
ACUDA (저수준 직접 작성)
BOpenACC (지시자 기반)
CMPI (분산 처리)
DPython NumPy
정답: B — OpenACC
OpenACC는 지시자(pragma/directive) 기반이므로 코드 구조를 크게 바꾸지 않고 빠르게 GPU에 올릴 수 있습니다. 발표자는 "먼저 최대한 많이 GPU에 올리고, 이후 프로파일러로 병목을 찾아 CUDA로 정밀 최적화하라"고 조언했습니다. 해당 사례 코드에서도 GPU 커널의 97%가 OpenACC로 커버됐고 CUDA는 3%에 불과했습니다.

발표자는 오히려 반대라고 강조했습니다. 더 빠른 시뮬레이션 = 더 많은 학습 데이터 생성 = 더 좋은 AI 모델이라는 선순환이 만들어지므로, AI 시대에도 시뮬레이션 최적화는 계속 중요합니다.

객관식 보통
반복 정제(Iterative Refinement) 혼합 정밀도 알고리즘의 순서로 올바른 것은?
AFP32 입력 → FP64 계산 → FP32 보정
BFP64 입력 → 저정밀도 계산(LU 분해) → FP64 복귀 후 잔차 반복 보정
CINT8 입력 → FP64 계산 → INT8 출력
DFP64 입력 → FP64 계산 → 오차 제거 없이 출력
정답: B
64비트 입력을 받아 낮은 정밀도로 내려 LU 분해를 빠르게 수행한 뒤 FP64로 되돌아와 잔차(residual)를 반복적으로 계산해 필요한 정확도에 도달할 때까지 정제합니다. 이 방식은 과거 HPC에서도 사용되던 검증된 기법입니다.

단답형 보통
NVIDIA의 Earth-2 프로젝트에서 기상 예측을 위해 활용된 수치기상예측(NWP) 코드의 이름을 쓰시오. (영문 약어)
정답: WRF (Weather Research and Forecasting)
WRF는 주로 Fortran으로 작성된 기상 연구 및 예측 NWP 모델로, MPI+OpenMP를 사용합니다. GPU 지원을 위해 OpenACC, std::par, CUDA 등 다양한 프로그래밍 모델이 혼용됩니다.

객관식 어려움
cuBLAS의 FP64 에뮬레이션에서 사용하는 텐서코어 유형과 수학적 체계(scheme)로 올바른 조합은?
ABF16 텐서코어 + Ozaki scheme
BINT8 텐서코어 + Ozaki scheme
CFP16 텐서코어 + LU 분해
DINT4 텐서코어 + Kahan summation
정답: B — INT8 텐서코어 + Ozaki scheme
발표자가 명시한 내용입니다. FP32 에뮬레이션은 BF16 텐서코어(IEEE 준수)를 사용하고, FP64 에뮬레이션은 INT8 텐서코어와 Ozaki scheme을 함께 사용합니다. Ozaki scheme은 부동소수점 연산을 정수 연산으로 정확하게 분해하는 수치 알고리즘입니다.

객관식 어려움
Q&A에서 "신세대 GPU가 구세대 대비 FP64 성능 향상 폭이 낮은 정밀도보다 작다"는 지적에 대해 발표자가 제시한 근본 이유는?
ANVIDIA가 FP64 유닛 개발에 투자를 줄였기 때문
B과학 코드들은 FP64 연산이 아닌 다른 부분(메모리·통신 등)에서 병목이 발생하므로, 균형 잡힌 전체 애플리케이션 성능을 기준으로 설계하기 때문
CFP64는 이미 완성된 기술이라 개선 여지가 없기 때문
D에뮬레이션으로 FP64를 대체하면 되기 때문
정답: B
발표자는 아키텍처를 설계할 때 애플리케이션 전체를 관점으로 삼는다고 밝혔습니다. 과학 코드들은 FP64 연산 자체보다 메모리 대역폭, 통신, I/O 등 다른 요인에서 병목이 발생합니다. 따라서 FP64 성능만 극단적으로 올리는 것보다 전체 성능 균형을 맞추는 쪽이 실제 애플리케이션 속도 향상에 더 효과적입니다.
